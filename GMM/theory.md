# The Big Picture (no math yet)

Read this slowly. Don't try to memorize anything. Just absorb the story.

### The setup

You have a bunch of points scattered on a number line. Looking at them, you can see they form **clusters** — some points near $-3$, some near $0$, some near $+4$. Your eye sees 3 clusters.

You want to write down a math formula that says: *"this data was generated by 3 Gaussian bell curves overlapping."* That formula is the **GMM**. You already know it:

> p(x) = π₁·N(x | μ₁, σ²₁) + π₂·N(x | μ₂, σ²₂) + π₃·N(x | μ₃, σ²₃)

Three bells, mixed with weights π that say "how big each bell is."

### The job

Find the right values for the bell parameters (the μ's, σ²'s, and π's) so that the formula matches your data well.

### The annoying problem

To find good bell parameters, it would help a lot to know **which bell each point came from**. If point $x = 3.2$ came from bell 3, I'd use it to update bell 3's mean. Easy.

But you don't know which bell each point came from. The data is just points — no labels.

**Chicken and egg:**
- Need parameters → know which bell each point belongs to.
- Need to know which bell → estimate parameters.

### The trick (this is EM, in plain words)

> *Guess the parameters. Use those guesses to figure out — softly — which bell each point probably came from. Use those soft guesses to update the parameters. Repeat until nothing changes.*

That's the whole algorithm. We'll fill in the math, but **that sentence is what you're learning**. Everything else is just bookkeeping for that sentence.

## The two unknowns

In a GMM, there are really **two different things you don't know**:

**Unknown 1: The bell parameters.**
- The means (where each bell sits): μ₁, μ₂, μ₃
- The variances (how wide each bell is): σ²₁, σ²₂, σ²₃
- The weights (how big each bell is in the mixture): π₁, π₂, π₃

**Unknown 2: Which bell each point came from.**
- For each point xᵢ, there's a hidden label zᵢ that says "I came from bell 2."
- You never see zᵢ. That's why it's called a **latent** (= hidden) variable.

That's it. Two kinds of unknowns. The bells, and the labels.

---

### Tiny check

If I have 100 data points and I'm fitting a 3-component GMM, how many **latent labels** (z values) are there in total?

#### 10

Exactly right. One z per data point.

You have 3 *bells* but 100 *labels*, because each of the 100 points needs to know which of the 3 bells it came from.

## The soft-label idea

Here's where GMM gets clever.

You might think: "okay, just guess a hard label for each point — say point 7 came from bell 2, done." That's actually what **k-means** does. Hard assignments. Each point belongs to exactly one cluster.

GMM does something softer.

Instead of saying *"point 7 came from bell 2"*, GMM says:

> *"Point 7 came from bell 1 with probability 0.05, from bell 2 with probability 0.85, and from bell 3 with probability 0.10."*

That's a **soft assignment**. The point doesn't fully belong to anyone — it's spread across all the bells, with most of its weight on the one that fits it best.

These three numbers (0.05, 0.85, 0.10) for point 7 are called the **responsibilities** for point 7. Symbol: **γ** (gamma).

We write:

> γ₇ = [0.05, 0.85, 0.10]

This is a row of length K = 3 (one entry per bell). The entries sum to 1 (because the point came from *some* bell, with total probability 1).

---

### One thing to notice

This is exactly what you saw in Phase 1 — the responsibility table. Each row was a point's soft labels. We just didn't fully name it back then.

When you had this table earlier:

```
x:    -1            0              1            7
γ:  [.15,.15,.7]  [.33,.33,.34]  [.25,.5,.25]  [1, 0, 0]
```

Row 1 says: *"Point x = -1 came from bell 1 with prob 0.15, bell 2 with prob 0.15, bell 3 with prob 0.7."*

Last row is special: γ = [1, 0, 0]. That point is **certain** it came from bell 1. Probability 1 on one bell, 0 on the others. That's a *hard* assignment hidden inside the soft-assignment framework.

## How do we compute γ?

You saw γ rows handed to you in a table. But in real life, **nobody hands you γ**. You compute it from the bell parameters.

Let's do this in plain words first, then add notation.

---

### The question γ answers

Point x = 2 just showed up. You have 3 bells, and you want to know:

> *"What's the probability this point came from bell k?"*

In symbols:

> **γᵢₖ = P(point i came from bell k, given that we observed xᵢ)**

In plain words: "given what I see, how likely is each bell?"

---

### How to compute it — the intuition

Two things matter for whether bell k "owns" point i:

**1. How well does bell k fit point i?**

If point i sits right on top of bell k's peak, bell k is a great explanation. If point i is far in the tails of bell k, bell k is a bad explanation.

That fit is measured by the bell curve value: **N(xᵢ | μₖ, σ²ₖ)**.

High value = good fit. Low value = bad fit.

**2. How big is bell k in the first place?**

If bell k has weight π = 0.01 (basically nonexistent in the mixture), then even if it fits the point well, it's unlikely to be the source. There's just not much of bell k around.

If bell k has weight π = 0.7 (most of the mixture is bell k), then it's a likely source by default.

---

### Combining them

For each bell k, the "evidence that bell k generated point i" is:

> **(weight of bell k) × (how well bell k fits point i)**
> = π_k × N(xᵢ | μₖ, σ²ₖ)

But this is just a raw score, not a probability. We need it to be a number between 0 and 1 that sums to 1 across all bells. So we **normalize** — divide by the total across all bells.

---

### The formula

$$\gamma_{ik} \;=\; \frac{\pi_k \cdot \mathcal{N}(x_i \mid \mu_k, \sigma_k^2)}{\sum_{j=1}^{K} \pi_j \cdot \mathcal{N}(x_i \mid \mu_j, \sigma_j^2)}$$

That's it. That's the responsibility formula.

**Plain reading:**

- **Numerator** = how much bell k claims point i (weight × fit, for bell k only).
- **Denominator** = how much **all bells together** claim point i (the same thing, summed across every bell).
- **Ratio** = bell k's share of the claim.

If bell k is the only good explanation, the ratio is close to 1.
If many bells fit equally well, the ratio is close to 1/K.
If bell k is a terrible fit, the ratio is close to 0.

Notice the denominator is just p(xᵢ) — the GMM density at point i, summed over all bells. So you can also write it as:

> γᵢₖ = (πₖ · bell k's value at xᵢ) / p(xᵢ)

Same thing, two ways to write it.

---

### Why the rows sum to 1

Sum γ over all k:

$$\sum_{k=1}^{K} \gamma_{ik} \;=\; \frac{\sum_{k=1}^{K} \pi_k \cdot \mathcal{N}(x_i \mid \mu_k, \sigma_k^2)}{\sum_{j=1}^{K} \pi_j \cdot \mathcal{N}(x_i \mid \mu_j, \sigma_j^2)} \;=\; \frac{\text{same thing}}{\text{same thing}} \;=\; 1$$

The top and bottom are the same sum — they cancel. Rows sum to 1 by construction. That's why you can always sanity-check a γ table.

---

### Worked numerical example (this is an actual exam question)

This is **exam_06072023** Q2(c), word for word.

**Given:**
- π = [0.57, 0.11, 0.22, 0.10]  (so K = 4 bells)
- For some point x, the bell values are:
  - bell 1: N(x | bell 1) = 0.01
  - bell 2: N(x | bell 2) = 0.25
  - bell 3: N(x | bell 3) = 0.05
  - bell 4: N(x | bell 4) = 0.50

**Compute γ for this point.**

---

**Step 1.** Multiply (weight × fit) for each bell. These are the numerators.

| k | πₖ | N(x \| bell k) | πₖ × N |
|---|------|---------|--------|
| 1 | 0.57 | 0.01 | 0.0057 |
| 2 | 0.11 | 0.25 | 0.0275 |
| 3 | 0.22 | 0.05 | 0.0110 |
| 4 | 0.10 | 0.50 | 0.0500 |

**Step 2.** Add them up. This is the denominator (same for all k):

$$0.0057 + 0.0275 + 0.0110 + 0.0500 = 0.0942$$

**Step 3.** Divide each numerator by the denominator:

| k | πₖ × N | ÷ 0.0942 | γₖ |
|---|--------|----------|------|
| 1 | 0.0057 | 0.0057/0.0942 | **0.061** |
| 2 | 0.0275 | 0.0275/0.0942 | **0.292** |
| 3 | 0.0110 | 0.0110/0.0942 | **0.117** |
| 4 | 0.0500 | 0.0500/0.0942 | **0.531** |

**Step 4.** Sanity check — sum should be 1:

$$0.061 + 0.292 + 0.117 + 0.531 = 1.001 \;\;\checkmark \text{ (rounding)}$$

**Answer:** γ = [0.061, 0.292, 0.117, 0.531]

Bell 4 dominates (0.531), even though bell 1 had the highest weight (0.57). Why? Bell 1's fit was terrible (0.01), so its raw score is tiny. Bell 4 had a moderate weight (0.10) but a great fit (0.50), so it wins.

This is the **trade-off γ captures**: weight × fit, normalized.

---

### The recipe — memorize this for the exam

> **To compute γ for a point:**
> 1. For each bell k, compute πₖ × (bell k's value at the point). Call this "score k."
> 2. Add all scores → "total."
> 3. γₖ = score k / total.
> 4. Check: γ row should sum to 1.

Three lines. That's the E-step.

---

## The M-step — updating the bell parameters

You just learned how to compute γ from the bell parameters. Now we do the reverse: given γ, update the bell parameters.

Plain words first:

> *We have soft labels for every point. Use them to update each bell's weight, mean, and variance.*

That's the M-step. ("M" stands for "Maximization." Ignore that name. It just means "update the parameters.")

---

### The intuition for each update

You have N points and K bells. For each bell k, you want three things: a new weight πₖ, a new mean μₖ, and a new variance σ²ₖ.

#### Update 1: How big should bell k be (πₖ)?

Bell k's "size" in the mixture should reflect how many points it owns. But the points are softly owned — point 1 owns 0.7 of itself to bell k, point 2 owns 0.1 of itself, etc.

So we add up all the soft ownerships:

> **N_k = γ₁ₖ + γ₂ₖ + ... + γ_Nk**

N_k is called the "effective number of points in bell k." It's the soft count — somewhere between 0 and N.

Then πₖ is just the *fraction* of total points that bell k owns:

> **πₖ = N_k / N**

That's it. Total ownership divided by total points.

**Sanity check:** the γ rows sum to 1, so if you add up all N_k across bells, you get N (every point contributes 1 total ownership). So Σ πₖ = N/N = 1. The weights still sum to 1.

---

#### Update 2: Where should bell k sit (μₖ)?

If bell k fully owned a set of points, its new mean would just be the average of those points. With soft ownership, it's a **weighted average**, where each point's weight is how much bell k owns it:

> **μₖ = (γ₁ₖ·x₁ + γ₂ₖ·x₂ + ... + γ_Nk·x_N) / N_k**

In summation notation:

$$\mu_k \;=\; \frac{1}{N_k}\sum_{i=1}^{N} \gamma_{ik} \cdot x_i$$

Compare to the regular sample mean: $\bar{x} = \frac{1}{N}\sum_i x_i$. Same shape — except instead of weighting every point equally with 1/N, we weight by γᵢₖ/N_k.

Points that bell k strongly owns (γ near 1) pull μₖ toward them. Points that bell k barely owns (γ near 0) barely affect μₖ.

---

#### Update 3: How wide should bell k be (σ²ₖ)?

Same idea: weighted variance. Take the squared distance of each point from the new mean μₖ, weighted by how much bell k owns the point:

$$\sigma_k^2 \;=\; \frac{1}{N_k}\sum_{i=1}^{N} \gamma_{ik} \cdot (x_i - \mu_k)^2$$

Compare to regular sample variance: $\frac{1}{N}\sum_i (x_i - \bar{x})^2$. Same shape, weighted by γ.

---

### The full M-step in one box

For each bell k:

```
N_k  = Σᵢ γᵢₖ                            (soft count)

πₖ   = N_k / N                          (weight)

μₖ   = (1/N_k) · Σᵢ γᵢₖ · xᵢ            (weighted mean)

σ²ₖ  = (1/N_k) · Σᵢ γᵢₖ · (xᵢ - μₖ)²    (weighted variance)
```

Four formulas. Always in the same order: N_k first (you need it for all three), then π, then μ, then σ² (which needs the new μ).

### Worked numerical example — this is a real exam question

From **ML_Jan_19_2026**, Q4(3). You've already seen this table.

**Given γ for 4 points (K = 3 bells):**

```
x:    -1            0              1            7
γ:  [.15,.15,.7]  [.33,.33,.34]  [.25,.5,.25]  [1, 0, 0]
```

**Compute the updated π.**

This is just N_k / N for each k. Let me do it column by column.

**Bell 1.** Add the first entry of each γ row:

$$N_1 = 0.15 + 0.33 + 0.25 + 1.00 = 1.73$$

**Bell 2.** Second entries:

$$N_2 = 0.15 + 0.33 + 0.50 + 0.00 = 0.98$$

**Bell 3.** Third entries:

$$N_3 = 0.70 + 0.34 + 0.25 + 0.00 = 1.29$$

**Sanity check.** N_1 + N_2 + N_3 should equal N = 4:

$$1.73 + 0.98 + 1.29 = 4.00 \quad\checkmark$$

**Now divide by N = 4:**

$$\pi_1 = 1.73 / 4 = 0.4325$$
$$\pi_2 = 0.98 / 4 = 0.2450$$
$$\pi_3 = 1.29 / 4 = 0.3225$$

**Final answer:**

$$\boxed{\pi = [0.4325,\ 0.2450,\ 0.3225]}$$

That matches answer **A** in the exam. The distractors are typical traps:
- **B** has 4 numbers, but π has K entries, not K+1.
- **C** is the unnormalized N_k vector (forgot to divide by N).
- **D** is uniform 0.25 (the prior, not the posterior).
- **E** has the right numbers but wrong order. *Order matters.*

⚠️ The order trap is the most common mistake — students compute the right numbers and rearrange them. Bell 1 is the first column of γ. Bell 2 is the second column. Bell 3 is the third. Don't sort.

### What the M-step looks like, in one English sentence

> *Each bell becomes a weighted summary of all data points, where the weights are that bell's soft ownerships.*

Big-ownership points pull the bell strongly. Tiny-ownership points are nearly ignored.

## Putting it together — the full EM loop

You now know both halves. Let me show you the whole algorithm in one place.

```
INITIALIZE:
  Pick starting values for π, μ, σ²  (random, or via k-means)

REPEAT until convergence:

  E-step:  for every point i, for every bell k:
            compute γᵢₖ = (πₖ · N(xᵢ|μₖ,σ²ₖ)) / (Σⱼ πⱼ · N(xᵢ|μⱼ,σ²ⱼ))

  M-step:  for every bell k:
            N_k = Σᵢ γᵢₖ
            πₖ  = N_k / N
            μₖ  = (1/N_k) Σᵢ γᵢₖ · xᵢ
            σ²ₖ = (1/N_k) Σᵢ γᵢₖ · (xᵢ - μₖ)²

STOP when:
  the parameters stop changing,
  OR the log-likelihood stops improving.
```

That's the entire EM algorithm. Six lines of pseudocode.

---

### Why does it work? (Two-sentence version)

> *The E-step gives the best soft labels you can compute given the current parameters. The M-step gives the best parameters you can compute given those soft labels.*

Each step improves the fit (specifically, each step never decreases the log-likelihood). Since the log-likelihood can't increase forever — it's bounded — the algorithm has to converge.

We're skipping the formal proof (it involves the ELBO and Jensen's inequality, and the exam doesn't ask for that derivation, only the recipe).

---

### What EM does *not* guarantee

**It does NOT always find the global best fit.** It finds *a* local optimum — meaning, a set of parameters where small changes make things worse. There might be a much better set somewhere else in parameter space, and EM won't find it from a bad starting point.

This is why **initialization matters**. Common strategies:

1. **Random initialization** — quick but unreliable, often gets stuck.
2. **K-means initialization** — run K-means first, use the centroids as initial μₖ. This is what sklearn does by default.
3. **Multiple restarts** — run EM many times from different starts, keep the best result.

⚠️ Exam-relevant: if asked *"why does EM converge?"* the answer is *"it monotonically increases the log-likelihood, which is bounded above, so it must converge — but to a local maximum, not necessarily a global one."*

## Sampling from a fitted GMM

This is a small but heavily-tested topic. Once you've fit a GMM (you have π, μ, σ² for each bell), you might want to **generate new fake data** from it. The exam asks this in **ML_662025_A Q3(5)** and **exam_23062022 Q2(e)**.

The recipe is **two steps per sample**:

### Step 1: Pick which bell

You have weights π = [π₁, π₂, ..., π_K] that sum to 1. Sample one bell at random, weighted by π.

How? Draw a uniform random number u in [0, 1], then look up which π-bucket it falls into.

- If π = [0.3, 0.5, 0.2] and u = 0.7:
  - Bucket 1: [0, 0.3)
  - Bucket 2: [0.3, 0.8)
  - Bucket 3: [0.8, 1.0]
  - u = 0.7 lands in bucket 2 → pick bell 2.

This is called **sampling from a Categorical distribution**.

### Step 2: Sample from the chosen bell

Once you've picked, say, bell 2, draw a sample from N(μ₂, σ²₂). That's a standard Gaussian draw.

How do you draw from N(μ, σ²)? You draw a standard normal sample z from N(0, 1), then transform: **x = μ + σ·z**.

(Drawing from N(0,1) is a primitive — `np.random.randn()` in NumPy. Behind the scenes it uses the Box-Muller transform or similar, but you treat it as given.)

---

### The two-step recipe in box form

```
TO SAMPLE ONE POINT FROM A FITTED GMM:

  1. Sample k ~ Categorical(π)
     Concretely: u ~ Uniform[0,1], then find which π-bucket u falls in.

  2. Sample x ~ N(μₖ, σ²ₖ)
     Concretely: z ~ N(0,1), then x = μₖ + σₖ · z.
```

You need **two random sources**: a uniform-in-[0,1] (to pick the bell) and a standard normal (to pick the point within the bell).

## The known-assignment case

This is the **easiest** case of GMM fitting. It shows up in **exam_23062022** and **ML_662025_A**, and you've actually already seen me solve one of these in Phase 2 — but now we'll do it properly, knowing the EM framework.

### The setup

Normally, z is hidden. You don't know which bell each point came from. That's the whole reason EM exists.

But sometimes, the exam **gives you z**. For example:

$$
\begin{array}{c|cccccccc}
x & 3 & -5 & 2 & -1 & -3 & -2 & 4 & -4 \\
\hline
z & 0 & 1 & 0 & 1 & 0 & 1 & 0 & 1
\end{array}
$$

Each point has a known label. Now, what happens to EM?

---

### What changes when z is known

When z is known, γ is no longer a soft probability. It collapses to **hard 0s and 1s**.

If point i has label z_i = k, then bell k owns it completely:

$$
\gamma_{ik} =
\begin{cases}
1 & \text{if } z_i = k \\
0 & \text{otherwise}
\end{cases}
$$

So the γ table is no longer something to compute — you can **write it down directly** from the z table.

### Why this matters

Look back at the M-step formulas:

$$
N_k = \sum_i \gamma_{ik}
$$

$$
\mu_k = \frac{1}{N_k} \sum_i \gamma_{ik} \cdot x_i
$$

$$
\sigma_k^2 = \frac{1}{N_k} \sum_i \gamma_{ik} \cdot (x_i - \mu_k)^2
$$

If γ is 0/1, the sums collapse:

- $N_k$ becomes just the **count** of points in bell k (every γ is 0 or 1; sum = count).
- $\mu_k$ becomes the **plain average** of points in bell k (only γ = 1 points contribute, with weight 1).
- $\sigma_k^2$ becomes the **plain variance** of points in bell k.

In other words, **with known z, you just do per-bell sample statistics**. No iteration. No E-step. One-shot computation.

This is what was hidden inside Phase 2's known-assignment example. Now you see why it worked: γ became hard, so EM collapsed to plain averaging.

---

### The closed-form recipe

For each bell k:

$$
N_k = \text{count of points where } z_i = k
$$

$$
\pi_k = \frac{N_k}{N}
$$

$$
\mu_k = \frac{1}{N_k} \sum_{i : z_i = k} x_i
$$

$$
\sigma_k^2 = \frac{1}{N_k} \sum_{i : z_i = k} (x_i - \mu_k)^2
$$

The notation $\sum_{i : z_i = k}$ means "sum over only those points whose label is k."

This is **single-Gaussian MLE applied independently to each group**. Nothing fancy.

---

### Worked example — ML_662025_A Q3(3)

This is the same example I did in Phase 2, but now you understand *why* it works (γ is hard).

Data:

$$
\begin{array}{c|cccccccc}
x & 3 & -5 & 2 & -1 & -3 & -2 & 4 & -4 \\
\hline
z & 0 & 1 & 0 & 1 & 0 & 1 & 0 & 1
\end{array}
$$

**Step 1. Group by z.**

- $z = 0$: points $\{3, 2, -3, 4\}$, count $N_0 = 4$
- $z = 1$: points $\{-5, -1, -2, -4\}$, count $N_1 = 4$

**Step 2. Compute the mixing weights.**

$$
\pi_0 = \frac{N_0}{N} = \frac{4}{8} = 0.5
$$

$$
\pi_1 = \frac{N_1}{N} = \frac{4}{8} = 0.5
$$

**Step 3. Compute the means.**

$$
\mu_0 = \frac{3 + 2 + (-3) + 4}{4} = \frac{6}{4} = 1.5
$$

$$
\mu_1 = \frac{-5 + (-1) + (-2) + (-4)}{4} = \frac{-12}{4} = -3.0
$$

**Step 4. Compute the variances.**

For bell 0, deviations from $\mu_0 = 1.5$:

$$
(3 - 1.5)^2 = 2.25
$$

$$
(2 - 1.5)^2 = 0.25
$$

$$
(-3 - 1.5)^2 = 20.25
$$

$$
(4 - 1.5)^2 = 6.25
$$

Sum = $2.25 + 0.25 + 20.25 + 6.25 = 29.0$

$$
\sigma_0^2 = \frac{29.0}{4} = 7.25
$$

For bell 1, deviations from $\mu_1 = -3.0$:

$$
(-5 - (-3))^2 = 4
$$

$$
(-1 - (-3))^2 = 4
$$

$$
(-2 - (-3))^2 = 1
$$

$$
(-4 - (-3))^2 = 1
$$

Sum = $4 + 4 + 1 + 1 = 10$

$$
\sigma_1^2 = \frac{10}{4} = 2.5
$$

**Final answer:**

$$
\pi = [0.5,\, 0.5], \quad \mu_0 = 1.5,\, \sigma_0^2 = 7.25, \quad \mu_1 = -3.0,\, \sigma_1^2 = 2.5
$$

This matches option A on the exam.

---

### Two traps to remember

**Trap 1: MLE uses divide-by-N, not divide-by-(N–1).**

If you reflexively use the "unbiased" sample variance $\frac{1}{N_k - 1}$ from a statistics class, you'd get $\sigma_0^2 = 29/3 \approx 9.67$, which doesn't match any answer. MLE is biased. Always divide by $N_k$.

**Trap 2: Group BEFORE computing μ.**

Don't compute the overall mean of all 8 points and use it for both groups. Each bell has its own mean computed only from its own points.

## GMM vs K-Means

You already know k-means at the API level. We're going to look at it through the GMM lens, because the exam frames k-means as **a special case of GMM**.

### The story

K-means and GMM are doing the same kind of thing — finding clusters in data — but with different assumptions and different machinery.

| | **K-means** | **GMM** |
|---|---|---|
| What it outputs | Hard cluster assignment per point | Soft probabilities per point (γ) |
| Cluster shape | Spherical, equal size | Any Gaussian shape (ellipses) |
| Algorithm | Iterate: assign → update centroids | Iterate: E-step → M-step |
| Probabilistic? | No | Yes |
| Parameters per cluster | Just $\mu_k$ | $\pi_k, \mu_k, \Sigma_k$ |

---

### K-means as a special case of GMM

This is the conceptual punchline. Take a GMM and make these three restrictions:

1. **All bells have the same fixed variance** $\sigma^2 = \text{const}$ (spherical, equal).
2. **All bells have equal weights** $\pi_k = 1/K$.
3. **Force hard assignments**: instead of computing soft γ, snap each point to its single best bell. That is, set $\gamma_{ik} = 1$ for the bell with the highest score, and 0 for all others.

What you get is exactly k-means:

- **E-step becomes**: assign each point to the nearest centroid (because with equal $\sigma^2$ and equal $\pi$, "best bell" reduces to "closest mean" in Euclidean distance).
- **M-step becomes**: update each centroid to be the average of its assigned points (because with hard 0/1 γ, the weighted average becomes a plain average over the assigned group).

So:

> **K-means = GMM with spherical equal-variance bells + equal weights + hard assignments.**

You can think of k-means as a stripped-down, faster, less flexible version of GMM.

---

### When does each fail?

This is what the exam likes to ask. Two failure modes per algorithm.

#### K-means fails when:

**1. Clusters have different sizes/shapes.**

If one cluster is a thin elongated stripe and another is a tight ball, k-means can't represent the difference — it assumes all clusters look the same (spherical, same size). It will mis-assign points near the boundary because Euclidean distance is its only tool.

**2. Clusters overlap heavily.**

Points in the overlap region get hard-assigned to one cluster, but they really belong to both. K-means can't express "this point is 60% cluster A, 40% cluster B." It commits, often wrongly.

#### GMM fails when:

**1. A single component collapses onto one data point.**

This is the **"variance collapse"** problem. If one bell's variance shrinks toward zero and centers on a single point, the likelihood for that point goes to infinity. The MLE is technically unbounded.

In practice, sklearn prevents this by adding a tiny floor ($\texttt{reg\_covar}$). Without that, EM crashes or gives nonsense.

**2. Wrong K.**

If you tell GMM K = 5 but the data really has 3 clusters, EM will still fit 5 bells. Two of them will end up degenerate (tiny weight, or overlapping with another). You need model selection (AIC/BIC) to choose K properly.

---

### The exam's two trick scenarios

These come from **exam_06072023**:

> *"Your classmate fits k-means with $k = 1$ for digit classification..."*

If $k = 1$, every point is in the same cluster. Every prediction is the same label (the mode of the training labels). **Useless classifier — always predicts the majority class.**

> *"...now consider $k \approx N$, so each cluster has roughly one point."*

If each cluster has one training point and you predict based on the nearest centroid's label, that's literally **1-Nearest-Neighbor** classification. The two algorithms become equivalent at this extreme.

The lesson: **k controls a bias–variance trade-off in k-means classification.** Small $k$ = high bias (oversimplifies). Large $k$ = high variance (memorizes training data).

## Step 1: Make a toy dataset

We'll generate data from a known GMM (two bells) so we can check whether our algorithm recovers the truth.We have 200 points. We *know* the truth (π = [0.4, 0.6], μ = [-3, 4], σ² = [1, 2]) but our EM algorithm will only see the points `X` and have to figure it out.

---

## Step 2: A single Gaussian density function

Before EM, we need a helper: given a point and a bell's parameters, compute the bell value $\mathcal{N}(x \mid \mu, \sigma^2)$.

Recall the formula:

$$
\mathcal{N}(x \mid \mu, \sigma^2) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(x - \mu)^2}{2\sigma^2}\right)
$$Works. At the peak (x = μ), the density is $1/\sqrt{2\pi} ≈ 0.3989$. Far from the mean, it decays to essentially zero. Standard bell behavior.

---

## Step 3: The E-step — compute γ

This is the heart of EM. For every point i and every bell k, compute:

$$
\gamma_{ik} = \frac{\pi_k \cdot \mathcal{N}(x_i \mid \mu_k, \sigma_k^2)}{\sum_{j=1}^{K} \pi_j \cdot \mathcal{N}(x_i \mid \mu_j, \sigma_j^2)}
$$

In code, we want γ to be a matrix of shape `(N, K)` — one row per point, one column per bell.Read this output carefully — it shows EM "thinking."

- Point x = -1 → γ = [0.622, 0.378, 0.000]. Bell 0 (centered at -1) is the best fit. Bell 2 (centered at 7) is so far away its responsibility is essentially zero.
- Point x = 7 → γ = [0.000, 0.000, 1.000]. Bell 2 owns it completely. (Familiar from the exam table!)
- All rows sum to 1. ✓

**This is exactly what the exam asks you to compute by hand**, and now you've written the code that does it.

---

## Step 4: The M-step — update π, μ, σ²

Recall the formulas:

$$
N_k = \sum_i \gamma_{ik}, \qquad \pi_k = \frac{N_k}{N}
$$

$$
\mu_k = \frac{1}{N_k} \sum_i \gamma_{ik} \cdot x_i, \qquad \sigma_k^2 = \frac{1}{N_k} \sum_i \gamma_{ik} \cdot (x_i - \mu_k)^2
$$**π = [0.4325, 0.2450, 0.3225]** — exact match with the exam answer A from ML_Jan_19_2026.

You just wrote the code that produces the exam's correct answer. If you ever doubt your by-hand computation, this is the reference.

Two extra observations on the new μ:

- Bell 1's new mean is **4.10** because the point x = 7 (with γ = 1 for bell 1) pulls the mean strongly to the right.
- Bell 1's new variance is **11.71** — huge, because bell 1 owns both x = -1 (γ = 0.15) and x = 7 (γ = 1.00), which are very far apart.

This is EM doing its job: γ told us bell 1 owns wildly separated points, so bell 1's variance has to grow to "explain" them all.

---

## Step 5: The full EM loop

Now glue it together. Initialize, loop until convergence.Look at this carefully. **Three things to notice:**

### 1. The log-likelihood goes up every iteration

That last check confirms it: `All log-likelihood increases non-negative? True`. The minimum increase is `3.04e-08` — essentially zero at convergence, but **never negative**. This is the monotonicity guarantee we talked about. You wrote it; it worked.

### 2. The algorithm recovered the truth — but with bells swapped

- True μ = [-3, 4]. Estimated μ = [4.095, -3.000]. ✓ but swapped order.
- True π = [0.4, 0.6]. Estimated π = [0.555, 0.445]. ✓ but swapped.

**This is called the label-switching problem.** EM doesn't know which bell is "bell 0" and which is "bell 1." The labels are arbitrary. Bell 0 in the truth corresponds to bell 1 in our estimate, and vice versa. Both fits are equally good — they describe the same density.

This is a known issue in GMM, not a bug. If you reorder estimates by mean:
- Truth (sorted): μ = [-3, 4], π = [0.4, 0.6]
- Ours (sorted): μ = [-3.000, 4.095], π = [0.445, 0.555]

Very close. The fit is genuinely good.

### 3. Convergence took 26 iterations

In the early iterations, the means are barely moving (both stuck around 0-2). Around iteration 15-20, they suddenly snap into place. That's typical EM behavior: a slow warmup, then a rapid commitment to a configuration.

## Step 6: Compare with sklearn

The acid test. Does sklearn give us the same answer?**Identical results.**

```
              pi              mu             var
Ours:    [0.445, 0.555]   [-3.000, 4.095]   [1.053, 1.516]
```

Our 50-line implementation matches sklearn to the third decimal. That's not a coincidence — sklearn does the **exact same math**. The differences in real life would be:

1. **Speed**: sklearn uses vectorized BLAS, our loop is slower for large N.
2. **Numerical stability**: sklearn works in log-space (log-sum-exp), so it doesn't underflow in high dimensions. Our version is fine for 1D.
3. **Initialization**: sklearn uses k-means by default; we used random data points. Both work here.
4. **The reg_covar trick**: sklearn adds 1e-6 to the variance to prevent collapse. We added the same `np.maximum(var, 1e-6)` floor.

```python
def gaussian_1d(x, mu, var):
    return (1/sqrt(2*pi*var)) * exp(-0.5*(x-mu)**2 / var)

def e_step(X, pi, mu, var):
    # gamma[i,k] = pi_k * N(x_i|mu_k,var_k) / sum_j (pi_j * N(...))
    for k: gamma[:,k] = pi[k] * gaussian_1d(X, mu[k], var[k])
    gamma /= gamma.sum(axis=1, keepdims=True)
    return gamma

def m_step(X, gamma):
    N_k = gamma.sum(axis=0)
    pi  = N_k / N
    mu  = (gamma.T @ X) / N_k
    var = [(gamma[:,k] * (X - mu[k])**2).sum() / N_k[k] for k]
    return pi, mu, var

def fit_gmm(X, K):
    initialize pi, mu, var
    loop:
        gamma = e_step(...)
        pi, mu, var = m_step(...)
        check log-likelihood convergence
```

---

**[2] Key implementation details**

- $\gamma$ has shape **(N, K)**: rows = points, cols = bells.
- $N_k$ is the **column sum** of γ (sums across all points, for each bell).
- `(gamma.T @ X) / N_k` is the vectorized form of $\mu_k = \frac{1}{N_k}\sum_i \gamma_{ik} x_i$.
- Floor variance with `np.maximum(var, 1e-6)` to prevent collapse.

---

**[3] Things you can verify yourself**

- Log-likelihood is non-decreasing every iteration (monotonicity).
- Each γ row sums to 1.
- Sum of $N_k$ across bells equals $N$.
- Final params match sklearn to ~3 decimals.

---

**[4] Why our code matches sklearn exactly**

Sklearn does the same EM, just:
- Vectorized for speed.
- Computed in log-space (log-sum-exp) for numerical stability in high dimensions.
- Uses k-means initialization.
- Adds `reg_covar` (variance floor) to prevent collapse.

The math is **identical**.

---

**[5] The label-switching "problem"**

EM doesn't know which bell is "bell 0." Estimates may come out reordered relative to the truth. This is not a bug. Sort bells by mean for comparison.
